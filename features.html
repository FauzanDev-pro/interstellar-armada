Implemented features

Graphics

  • cubemapped background
  • loading 3D models from XML
  • texture mapping with arbitrary color modulation
  • simple shading with directional lights
  • vertex luminosity data
  • models with multiple Level Of Detail (LOD)
  • support for line and triangle based drawing
  • camera facing and 1 axis fixed, turning billboards
  • multiple cameras with adjustable FOV
  • hierarchic mesh structure
  • frustum culling of meshes to optimize performance
  • alpha blending using vertex color data and transparent png textures (*only for billboards yet)
  • loading shader configuration from an XML file, easy shader setup in the program source

Physics

  • simplified newtonian physics allowing spaceship maneuvering in a way that is more realistic for space environment than presented in most movies or games

Game logic

  • loading definition of in-game object classes from XML (including automated load and setup for referenced model, texture and shader files)
  • loading full level configuration from XML (*excluding lighting information and cameras for now)
  • possibility to specify equipped weapons on individual spaceships

Gameplay

Planned features

Major planned features of the game

  • A game that is with you everywhere

    The game should run fluently and provide an appropriate (if modest) graphical experience from phones to desktop PCs. You can pick up where you left off on the device that is with you at the moment.

  • Realistic space battles

    Using (more) realistic, inertial combat lets you discover and leverage new dogfight maneuvers and tactics and provides a more unique but just as fun experience.

  • A vast world to explore

    Using powerful, smart and sophisticated random generation methods, lots of quests and places give you a long lasting immerse experience without becoming repetitive.
